Phase 4 — Transfer attack matrix · Week 8
Low compute cost — reuses already-generated adversarial examples
Take adversarial examples from P1 (image & audio), test against all other models in same modality. Compute TAR matrix. Expected finding: CNN→ViT transfer is low; CNN→CNN transfer is high. Produces Figure F3.
Phase 5 — Defense evaluation · Weeks 9–10
12 (model, defense) pairs — 3 brittle models × 4 defenses × 3 modalities
Apply JPEG/MP3 preprocessing, randomized smoothing, feature denoising (HGD), and PGD-AT adversarial training to the 3 most brittle models per modality. PGD-AT is most expensive (7-step inner maximization). Compute DRAT per pair. Build practitioner recommendation table (§6.4).
Phase 6 — Writing & submission · Weeks 11–12
5 figures drafted by week 10 — final sprint
All figures should be drafted during experiment phases. Final writing: §1 intro, §2 related work, §6 discussion are most time-consuming. Add 95% confidence intervals to all RAR values week 11. Statistical significance testing (3 seeds per experiment). Format for CISCOM special session.
Rule: Write one paragraph of the paper every single day you run experiments. By week 10 the paper must be 70% written — not 0%.

ε budget schedule (imperceptibility gate)
Image (L∞ norm)
ε ∈ {1, 2, 4, 8}/255. Gate: SSIM ≥ 0.95 on every accepted adversarial example. Discard failures — do not count them as successful attacks.
Audio (SNR-equivalent)
SNR values equivalent to image ε budgets. Gate: PESQ ≥ 3.5 (voice quality score). Psychoacoustic attack additionally requires masking threshold check.
Video
Frame-level SSIM ≥ 0.95 AND temporal consistency check (optical flow deviation < threshold). Temporally consistent PGD enforces this by design.
Post-compression pipeline (P2)
JPEG QF=75, H.264 CRF=23, social media sim (resize+compress+re-encode). Run all 3 on every adversarial example. Compute PCR for each.
Custom attack implementations (build these in week 1)
1
Frequency-targeted attack
PGD with gradient masked to DCT high-frequency coefficients only. Apply DCT → zero out low-freq gradients → inverse DCT → PGD step. Tests F3-Net specifically. ~100 lines of PyTorch.
2
Temporally consistent PGD
PGD with optical-flow warping between frames so perturbation field moves consistently with the face. Use OpenCV Farneback flow. Prevents frame-by-frame flicker artifacts that would violate imperceptibility. ~200 lines.
3
Asymmetric cross-modal attack
For BA-TFD and the late-fusion baseline: freeze video stream, compute gradient on audio only → PGD on audio. Then vice versa. Then joint. Compare RAR across 3 conditions. ~50 lines wrapping existing attack.

Core principle — preprocess once, reuse everywhere
Upload preprocessed data as a private Kaggle dataset. All 7 accounts import it.
This is the single highest-ROI action you can take. The active benchmark now runs from the Kaggle-mounted LAV-DF dataset, so preprocessing means generating reusable manifests and cached derivatives once, then reusing them across all accounts. Preprocess once → publish once → reuse everywhere.
Image preprocessing pipeline (do first, week 1)
1
Face detection + alignment
Use RetinaFace or MTCNN against the FF++ c23 clips mounted from Kaggle. Detect faces in sampled frames, align to canonical 224×224 crops, and cache only the derivatives you actually reuse across notebooks.
2
Frame sampling
Sample 10 frames per clip uniformly from the FF++ c23 primary split. Keep sampling deterministic so image baselines and attacks are reproducible across Kaggle sessions.
3
Manifest + cache creation
Write modality manifests first, then optionally cache face crops or frame tensors in LMDB or parquet-style shards. Keep the CSV manifest as the source of truth for every runner.
4
Landmark extraction for LipForensics
LipForensics requires 68 facial landmarks per frame. Use FAN (Face Alignment Network). Save as .npy files alongside face crops. This is LipForensics-specific but only needs to run once.
Audio preprocessing pipeline
1
Format normalization
Read each ASVspoof 2019 LA utterance directly from the Kaggle mount, resample to 16 kHz mono if needed, and cache only if repeated experiments make the decode cost worthwhile.
2
Pre-compute mel spectrograms
For LCNN and BA-TFD audio branches, pre-compute log-mel tensors for the active split when you need faster repeated attack runs. Save them as reusable `.npy` tensors keyed by manifest sample id.
3
Segment standardization
Pad or trim all ASVspoof 2019 LA utterances to the fixed 4 second budget used by AASIST and RawNet2 so batch shapes stay stable inside Kaggle notebooks.
Video preprocessing pipeline
1
Frame extraction + face cropping
Extract FF++ c23 clips at 8 fps for the video attack runners. Apply the same face crop strategy as the image path and cache sequences only when repeated evaluation outweighs direct decode from Kaggle input storage.
2
Clip segmentation
Use 30 second clips (or full video if shorter). For temporally consistent PGD, pre-compute optical flow or temporal consistency hints only for the subset you actually attack, because that cache is the expensive part.
Multimodal preprocessing (FakeAVCeleb primary, LAV-DF cross test)
Split audio and video tracks from the same FakeAVCeleb clip only when the multimodal attack pipeline needs cached derivatives. Reuse the same machinery for the LAV-DF cross-dataset clean test, keeping synchronization anchored to the original file path stored in metadata.
Time saved by preprocessing upfront: Estimated 40–60 GPU-hours saved across 7 accounts. Pre-computed LMDB + pre-aligned faces cuts per-experiment data loading from minutes to seconds.
